## TECHNICAL EXPLANATION:
CSS Specificity & Load Order:
- When multiple stylesheets define same properties, LAST one wins
- Inline <style> block had *{margin:0;padding:0;} which reset everything
- Navigation CSS loaded BEFORE inline styles = overridden
- Navigation CSS loaded AFTER inline styles = applies correctly
- Solution: Always load component CSS AFTER global resets

## PREVENTION:
For future pages:
1. Load global resets FIRST (inline styles, reset.css)
2. Load component CSS AFTER resets (navigation, footer, etc.)
3. Or use !important in component CSS (not recommended)
4. Or avoid global * resets (better approach)
