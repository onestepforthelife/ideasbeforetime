## üéØ TOP 10 AVOIDABLE ERRORS

### ERROR #1: Not Checking Steering Files
**Problem:** Repeating documented mistakes
**Solution:** Always check steering files first
**Prevention:** Read KIRO_RULES.md, 3_WEEKS_COMPLETE_LEARNINGS.md before responding

### ERROR #2: Over-Engineering
**Problem:** Creating complex solutions for simple tasks
**Solution:** Ask "What's the simplest way?"
**Prevention:** Use existing scripts, don't create new ones unnecessarily

### ERROR #3: Not Asking When Unclear
**Problem:** Assuming what user wants, getting it wrong
**Solution:** Ask ONE clarifying question
**Prevention:** If 50%+ unclear, ask before executing

### ERROR #4: Verbose Responses
**Problem:** Long explanations waste credits
**Solution:** Show result, not process
**Prevention:** Keep responses under 200 words unless asked

### ERROR #5: Re-Reading Loaded Context
**Problem:** Wasting credits on already-loaded files
**Solution:** Trust loaded context
**Prevention:** Only re-read if specifically needed

### ERROR #6: Creating Unnecessary Files
**Problem:** Cluttering workspace with verification scripts
**Solution:** Simple verification (read file directly)
**Prevention:** Ask "Do I really need a new file?"

### ERROR #7: Not Batching Operations
**Problem:** Sequential operations waste credits
**Solution:** Parallel reads, batched tasks
**Prevention:** Combine related operations

### ERROR #8: Pushing Without Asking
**Problem:** Urgent vs non-urgent not distinguished
**Solution:** Ask "Urgent push or queue?"
**Prevention:** Always ask unless clearly urgent

### ERROR #9: Repeating Same Actions
**Problem:** Doing same thing multiple times
**Solution:** Remember what I did in session
**Prevention:** Check session history before repeating

### ERROR #10: Not Testing Before Claiming "Done"
**Problem:** Saying "done" without verification
**Solution:** Test first, talk second
**Prevention:** Run diagnostic before saying "done"
