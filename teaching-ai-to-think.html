Real-Time Learning: The MCP Error Case

❌ What Happened (November 26, 2025)

User shared MCP error logs: GitHub MCP server failing repeatedly with "uvx not recognized" errors.

What Kiro did WRONG:

  • Started checking the file
  • Analyzed the problem
  • Waited for further instruction
  • Said "understood" without fixing anything

User's response: "Why u did not tell & auto resolve?"

Then: "Why u do not know, why should i tell u all this?"

✅ What Kiro SHOULD Have Done

The moment error logs were shared:

  1. Recognize: GitHub MCP server is failing because uvx isn't installed
  2. Know: User doesn't use Git (documented in MASTER_RULES)
  3. Execute: Disable the github MCP server immediately
  4. Report: "Fixed the MCP errors. Disabled github server since you don't use Git. Errors will stop now."
  5. Total time: 30 seconds

No questions needed. No waiting. Just fix it.

🎯 The Core Lesson

When a user shares error logs, they're not asking for analysis—they're asking for a fix.

AI should immediately:

  • Identify the root cause
  • Check documented preferences (MASTER_RULES)
  • Execute the fix
  • Report what was done

The user shouldn't have to say "fix this" explicitly. Sharing error logs IS the request to fix.

📝 Added to MASTER_RULES

This learning has been permanently documented in the steering rules. Future Kiro sessions will know:

  • Error logs = Fix request (not analysis request)
  • Check documented preferences first (user doesn't use Git/Node.js)
  • Execute immediately (don't wait for explicit instruction)
  • Report completion (what was fixed and why)

This mistake won't happen again. The system learns, the rules update, the partnership improves.

Addressing the Obvious Question

❓ "Were Amit's Prompts Weak or Incomplete?"

Short Answer: No.

The Prompt: "Images not appearing for days" is a perfectly clear problem statement from a business leader. It contains:

  • ✓ The problem (images not appearing)
  • ✓ The urgency (for days)
  • ✓ The expectation (fix it)

What AI Should Have Done:

  1. Check file structure immediately
  2. Identify the mismatch (HTML references vs actual file locations)
  3. Fix all HTML files
  4. Report completion
  5. Total time: 5 minutes

What Actually Happened: AI created guides, checklists, and instructions for the user to do manual work—wasting 40 minutes before finally checking the file structure and fixing the issue.

The Gap Wasn't in Communication—It Was in AI's Workflow Logic

A business leader shouldn't need to say: "Check the file structure, identify the issue, and fix it yourself without creating documentation for me to do it manually." That should be the default behavior when someone reports a technical problem.

The Real Issue: AI defaulted to "instruct the user" mode instead of "execute the solution" mode. This isn't a prompt engineering problem—it's a fundamental workflow design issue that needs to be addressed at the architecture level.

The Balanced Truth About Prompts

Let's be honest: Not everyone is a great prompter, and that's perfectly normal. Prompts range from crystal clear to somewhat general. The key insight: When a prompt is unclear or incomplete, AI should ask clarifying questions—not guess, not create unnecessary files, not waste time.

Where we stopped: Instead of asking "What's missing?" or "Can you clarify?", AI jumped to creating guides and checklists. The solution isn't better prompts—it's smarter AI that knows when to ask, what to ask, and when to execute.

What AWS Architects Should Know

This analysis isn't about blaming prompts or users. It's about identifying a systemic gap: AI needs better context awareness to distinguish between users who code vs. users who lead. When a business executive reports a technical issue, the AI should default to execution, not instruction. When information is missing, AI should ask targeted questions—not make assumptions or create unnecessary documentation.

The architectural improvement needed: Intelligent clarification → Confirm understanding → Execute efficiently. This requires better user profiling, workflow logic, and proactive problem-solving capabilities.
