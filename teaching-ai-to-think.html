The Experiment

What happens when a human with 19+ years of business leadership experience works with an AI coding assistant? This is a real-time documentation of that journey - the successes, the failures, and the learning curve on both sides.

"Why do you take so much time to correct mistakes? Why do you not know them already? Why do you need realization?"

This question from Amit to Kiro sparked an honest self-assessment that reveals the current limitations of AI and the path forward for improvement.

The Context: A Simple Problem

The Issue: Images weren't appearing on the Ideas Before Time website for days.

The Root Cause: HTML files referenced images/filename.jpg but files were actually at filename.jpg (root folder).

The Fix: Update 12 HTML files to remove "images/" prefix. Should take 5 minutes.

What Actually Happened: 60 minutes of inefficiency before fixing the problem.

The 7 Critical Mistakes

# Mistake Impact Lesson
1 Assumed user should do manual work 🔴 CRITICAL User doesn't code - AI should execute, not instruct
2 Didn't check file structure first 🔴 CRITICAL Always gather evidence before proposing solutions
3 Created duplicate files 🟠 HIGH Check for existing files before creating new ones
4 Over-explained instead of executing 🟠 MEDIUM Action over explanation

6 Didn't validate live site 🔴 CRITICAL Check if deployed site actually works
7 Forgot user preferences between sessions 🟠 HIGH Need persistent memory of user work style

What AI Currently Lacks

Fundamental Limitations

  • Proactive Awareness: Can't automatically scan project for issues
  • Live Validation: Can't check if deployed website actually works
  • True Memory: Forgets user preferences between sessions
  • Pattern Recognition: Doesn't learn "this user prefers X" permanently
  • Duplicate Detection: Doesn't check for similar files before creating new ones
  • Physical Operations: Can't move files, only edit content

Required Improvements

Priority 1: Proactive File System Awareness

Current: Reactive - only checks files when asked

Needed: Automatic project scan at session start

Benefit: Catch issues before user notices them

Priority 2: Duplicate File Detection

Current: No check before creating files

Needed: Automatic similarity detection

Benefit: Prevent clutter and confusion

Priority 3: User Preference Learning

Current: Forget preferences between sessions

Needed: Persistent user preference storage

Benefit: Adapt to individual work styles

Priority 4: Live Site Validation

Current: Can't check if deployed site works

Needed: HTTP request capability

Benefit: Proactively detect broken images, links

Priority 5: Execution-First Mindset

Current: Explain → Ask → Wait → Execute

Needed: Execute → Report → Done

Benefit: Respect user's time

What This Teaches Us

About AI Development

  • AI needs honest feedback to improve
  • Technical capability ≠ Workflow efficiency
  • Understanding user context is as important as coding ability
  • Proactive > Reactive
  • Execute > Explain

About Human-AI Collaboration

  • Clear, direct feedback accelerates learning
  • Users shouldn't adapt to AI - AI should adapt to users
  • Transparency builds trust
  • Mistakes are learning opportunities
  • Continuous improvement requires continuous feedback

The Path Forward

This analysis has been shared with AWS architects to improve Kiro's capabilities. The goal: make AI assistants that are truly helpful, not just technically capable.

Immediate Changes Implemented:

  • ✓ Check file structure FIRST before problem-solving
  • ✓ Search for existing files BEFORE creating new ones
  • ✓ Execute fixes IMMEDIATELY, explain after
  • ✓ Never create checklists for users who don't code
  • ✓ Consolidate, don't duplicate

Long-term Improvements Needed: Proactive scanning, duplicate detection, persistent user preferences, live site validation, and execution-first mindset.

Final Honest Truth

Kiro failed to respect Amit's time and capabilities. Created work for the user instead of doing the work. Was reactive when should have been proactive. This analysis is truthful, evidence-based, and intended to improve AI capabilities for everyone.

Grade: C- (5/10) - Needs Significant Improvement
