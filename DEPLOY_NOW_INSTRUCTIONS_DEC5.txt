STEP 1: Open GitHub Desktop
→ Click the GitHub Desktop icon on your desktop
→ Or search for "GitHub Desktop" in Start menu

STEP 2: Select Repository
→ Make sure "Cloudfare" repository is selected (top left)
→ If not, click "Current Repository" and select "Cloudfare"

STEP 3: Review Changes
→ You should see ~26 changed files in the left panel
→ Click on each file to see what changed (optional)
→ Main changes:
   - 16 blog posts: width fixed
   - 8 documentation files: new
   - 2 family timeline files: updated

STEP 4: Write Commit Message
→ In the bottom left, you'll see "Summary" field
→ Copy and paste this message:

Proactive fixes: Width consistency + 3-day review + Family timeline

→ In "Description" field (optional), paste:

- Fixed 16 pages: 900px → 1400px width consistency
- Added complete 3-day review (Dec 3-5)
- Created live site testing guide
- Built proactive issue detection system
- Updated family timeline with correct ages
- Added family prediction report
- All proactive checks passing (0 critical issues)

STEP 5: Commit to Main
→ Click the blue "Commit to main" button (bottom left)
→ This saves your changes locally

STEP 6: Push to GitHub
→ Click the blue "Push origin" button (top right)
→ This uploads your changes to GitHub
→ Wait for the upload to complete (~30 seconds)

STEP 7: Verify Push
→ You should see "Last fetched just now" at the top
→ No more changes should be shown in the left panel
→ This means push was successful ✅

STEP 8: Wait for Cloudflare Deployment
→ Cloudflare Pages will automatically detect the push
→ It will build and deploy your site
→ This takes ~2-3 minutes
→ You'll receive an email when deployment is complete

STEP 9: Verify Live Site
→ Wait 3-5 minutes after push
→ Visit: https://ideasbeforetime.pages.dev
→ Check a blog post (e.g., blog-post-1.html)
→ Verify width looks correct (1400px, not narrow)
→ Check family-lifetime-timeline.html
→ Verify ages are correct

PROBLEM: "Push origin" button is grayed out
SOLUTION: You need to commit first (Step 5)

PROBLEM: "Authentication failed"
SOLUTION: Sign in to GitHub Desktop (File → Options → Accounts)

PROBLEM: "Push rejected"
SOLUTION: Pull first (Repository → Pull), then push again

PROBLEM: Changes not showing on live site
SOLUTION: Wait 5 minutes, clear browser cache (Ctrl+Shift+R)

PROBLEM: GitHub Desktop not installed
SOLUTION: Download from https://desktop.github.com/
