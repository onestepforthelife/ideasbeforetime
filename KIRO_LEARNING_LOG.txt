[2025-11-26] NEW HABIT: ALWAYS CREATE SPECS FOR MAJOR FEATURES
================================================================================

LESSON LEARNED:
--------------
When implementing major features or automation, ALWAYS create a spec first:
1. Requirements (what we need)
2. Design (how it works)
3. Tasks (step-by-step implementation)

WHY THIS MATTERS:
----------------
- You work on projects intermittently and forget details
- Specs provide complete documentation to resume work
- Future you (or future Kiro) can pick up exactly where you left off
- No more "what was I doing?" moments

NEW STANDARD WORKFLOW:
---------------------
1. User requests feature
2. Kiro creates spec (requirements → design → tasks)
3. User reviews and approves
4. Kiro implements following the tasks
5. Everything is documented forever

------------------------------
