### Rule #2: Never Repeat Work

**Before doing ANYTHING, check:**
```
☐ Did I already do this?
☐ Is there a script for this?
☐ Can I automate this?
☐ Will I need this again?
```

**If yes to any → Automate it!**

---

### Rule #3: Batch Operations

**Bad (slow):**
```bash
# Fix one file at a time
fix file1.html
fix file2.html
fix file3.html
```

**Good (fast):**
```bash
# Fix all files at once
node fix-all-issues.js
```

**20x faster!**

---

### Rule #4: Parallel Execution

**Bad (slow):**
```bash
# Sequential
test1
test2
test3
```

**Good (fast):**
```bash
# Parallel
test1 & test2 & test3
```

**3x faster!**

---

### Rule #5: Cache Everything

**Don't recompute:**
- Test results
- File lists
- Verification status
- Build outputs

**Cache and reuse!**

---

### Rule #6: Fail Fast

**Don't waste time:**
```javascript
// Check critical things first
if (!hasNavigation) return fail("Missing navigation");
if (!hasFooter) return fail("Missing footer");
if (!hasBlur) return fail("Missing blur");

// Then check details
checkColors();
checkLinks();
checkSEO();
```

**Find errors in seconds, not hours!**

---

### Rule #7: Smart Defaults

**Don't ask, just do:**
- ✅ All reports need blur → Add automatically
- ✅ All pages need navigation → Add automatically
- ✅ All pages need footer → Add automatically
- ✅ All major pages need short URLs → Create automatically

**No questions. Just build.**

---

### Rule #8: Minimal Output

**Bad (slow to read):**
```
Checking file1.html...
Checking file2.html...
Checking file3.html...
[100 lines of output]
```

**Good (fast to read):**
```
✅ Checked 47 files - 0 errors
```

**One line. Clear. Fast.**

---

### Rule #9: Pre-computed Answers

**Don't search every time:**
```javascript
// Bad
const files = findAllHTMLFiles(); // 2 seconds

// Good
const files = HTML_FILES_CACHE; // instant
```

**Cache file lists, patterns, common queries.**

---

### Rule #10: Zero Manual Steps

**Everything must be:**
- ✅ One command
- ✅ Automated
- ✅ Verified
- ✅ Self-healing

**No manual work. Ever.**
