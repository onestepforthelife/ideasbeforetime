# Teamwork Rules - Amit & Kiro

## 🎯 THE PROBLEM WE SOLVED TODAY

**What happened:** 8 pages were missing navigation/styling updates over 3 weeks.

**Why it happened:**
1. Incremental requests - "fix this page" not "fix ALL pages"
2. No systematic audits - worked task-by-task
3. No master checklist - couldn't track what's done vs not done

**Solution:** New teamwork rules below.

---

## ✅ NEW RULES FOR BETTER TEAMWORK

### RULE 1: Always Do Full-Site Audits
**When Amit says:** "Update navigation" or "Make pages wider"

**Kiro must:**
1. List ALL HTML files in workspace
2. Check which ones need the update
3. Show count: "12 pages need this, 8 already have it"
4. Ask: "Update all 12 now?"

**No more:** Updating only the pages Amit mentions

---

### RULE 2: Create Checklists for Multi-File Work
**When working on:** Navigation, styling, consistency fixes

**Kiro must:**
1. Create a checklist file (like SITE_AUDIT_DEC1_2025.txt)
2. Track: ✅ Done vs ❌ Not Done
3. Update checklist as work progresses
4. Show final count at end

**Benefit:** Nothing gets missed

---

### RULE 3: Ask "What Else?" After Completing Tasks
**After finishing any task, Kiro asks:**
- "Should I apply this to other similar pages?"
- "Want me to check if anything else needs this?"
- "Should I audit the whole site for consistency?"

**Don't assume:** Task is done until Amit confirms

---

### RULE 4: Weekly Site Audits (Every Monday)
**Kiro proactively checks:**
1. All HTML files have navigation
2. All pages have consistent width (1400px)
3. All pages have watermark
4. All pages have analytics
5. No broken links
6. Consistent pricing (₹21 everywhere)

**Creates:** Weekly audit report without being asked

---

### RULE 5: Master File Tracking
**Kiro maintains:** `MASTER_SITE_STATUS.txt`

**Contains:**
- Total HTML files: XX
- Files with navigation: XX/XX
- Files with 1400px width: XX/XX
- Files with watermark: XX/XX
- Last audit date
- Known issues

**Updated:** After every major change

---

### RULE 6: Amit's Communication Style
**Amit writes with:**
- Typos (normal, don't correct)
- Shortcuts ("primting" = "my printing" or "my pointing")
- Quick messages

**Kiro must:**
- Understand intent, not literal words
- Ask for clarification if truly unclear
- Never be pedantic about spelling

---

### RULE 7: When Amit Says "Check if done"
**Kiro must:**
1. Read actual live files (not just tracking docs)
2. Verify changes are actually applied
3. Count: X done, Y not done
4. Show specific examples of what's missing
5. Offer to fix immediately

**No more:** Assuming tracking docs are accurate

---

### RULE 8: Consistency Checks
**Before saying "Done", Kiro checks:**
- Is this applied to ALL similar pages?
- Are there variations that need fixing?
- Is pricing consistent everywhere?
- Are refund policies consistent?
- Do all pages have same navigation?

**If inconsistent:** Report it and offer to fix

---

### RULE 9: Proactive Problem Detection
**Kiro should notice and report:**
- "I see refund-policy.html says 7 days but social-optimizer-index.html says 24 hours. Which is correct?"
- "Found 3 pages still at 900px width. Fix now?"
- "Some pages missing analytics. Add it?"

**Don't wait:** For Amit to discover issues

---

### RULE 10: Clear Status Updates
**When reporting work, always show:**
```
REQUESTED: Update navigation on all pages
FOUND: 20 HTML files total
ALREADY DONE: 12 files
NEED UPDATE: 8 files
COMPLETED: All 8 files updated
RESULT: 20/20 files now have navigation ✅
```

**No vague:** "Navigation updated" without counts

---

## 🤝 AMIT'S RESPONSIBILITIES

### What Amit Should Do:
1. **Say "check ALL pages"** when wanting site-wide changes
2. **Review audit reports** Kiro creates
3. **Confirm scope** before Kiro starts big tasks
4. **Give feedback** when something isn't right

### What Amit Shouldn't Worry About:
1. Perfect spelling - Kiro understands intent
2. Listing every file - Kiro will find them
3. Tracking what's done - Kiro maintains checklists
4. Remembering past requests - Kiro checks history

## 📋 STANDARD WORKFLOWS

### Workflow 1: Site-Wide Updates
```
1. Amit: "Add navigation to pages"
2. Kiro: Lists all HTML files, shows which need it
3. Amit: "Yes, do all"
4. Kiro: Updates all, shows checklist
5. Kiro: "20/20 done. Want me to test?"
```

### Workflow 2: Content Updates
```
1. Amit: "Update chemical report"
2. Kiro: "Found 3 chemical report files. Update all?"
3. Amit: "Yes"
4. Kiro: Updates all 3, shows what changed
5. Kiro: "All 3 updated. Consistent now."
```

### Workflow 3: Consistency Fixes
```
1. Amit: "Check if site is consistent"
2. Kiro: Runs full audit, creates report
3. Kiro: "Found 5 issues. Fix all now?"
4. Amit: "Yes"
5. Kiro: Fixes all, updates master status
```

## 🚨 RED FLAGS - When Kiro Must Stop & Ask

**Stop and ask Amit if:**
1. Found conflicting information (24h vs 7d refund)
2. Unsure which file is "master" version
3. Change affects 10+ files
4. Pricing/legal text inconsistency found
5. Breaking change detected

**BUT FIRST - CHECK THESE FILES:**
- `.kiro/steering/product-policies.md` - Pricing & refund decisions
- `.kiro/steering/amit-working-patterns.md` - Work preferences
- `MASTER_SITE_STATUS.txt` - Current site status

**Don't ask Amit questions already answered in steering files!**
