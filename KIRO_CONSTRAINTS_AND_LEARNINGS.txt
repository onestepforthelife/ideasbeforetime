### Git Push Loop Issue - Root Cause Analysis

**What Happened:**
Git pull --rebase got stuck asking "Should I try again? (y/n)" for deleting 'images' directory

**Why It Got Stuck:**
1. Remote had changes we didn't have locally (branches diverged)
2. Tried to rebase, which attempted to delete 'images' folder
3. Folder deletion failed (likely locked by another process)
4. Git entered interactive mode waiting for y/n input
5. PowerShell couldn't pipe input to the interactive prompt properly
6. Multiple attempts created a loop without resolving the stuck process

**Lesson Learned:**
- Always check git status before pushing
- Use force push with lease when branches diverge: `git push --force-with-lease`
- Kill stuck git processes immediately: `taskkill /F /IM git.exe`
- Don't try multiple approaches when process is stuck - kill it first
